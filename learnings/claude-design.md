# Claude Design でデザイン案を出すときの知見

初出: 2026-05-29

[Claude Design](https://claude.ai/design)（Anthropic の Web アプリ、Pro/Max/Team/Enterprise 向け）でデザイン案を生成し、Claude Code 側で実装する際のハマりどころ。

## チャット入力は改行（Enter）＝送信

- Claude Design のチャット入力欄は **改行（Enter）が送信トリガー**。複数行プロンプトを貼ると行ごとに分割送信され、意図が壊れる
- → プロンプトは**改行を含めず 1 通**で送る

## ブランドの読ませ方

- GitHub リポ連携、または `design-brief.md` のようなブランド要約をチャットに貼る
- カラートークン・フォント・ページ構成・トーンをまとめておくと精度が上がる

## 生成物の扱い

- 生成される HTML/CSS はそのまま使わず、**仕様（レイアウト・配色・モチーフ）として参照**し、Next.js + Tailwind + shadcn/ui のコンポーネント設計で実装し直す
- 「ソースコード自体がポートフォリオ」のような品質要件があるリポでは特に重要

## 操作を自動化する場合

- claude-in-chrome 等でブラウザ操作する場合、停止直後は UI が変化して入力先がずれることがある。送信前にスクリーンショットで入力欄を確認する
