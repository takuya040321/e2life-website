# サイトデザイン全面刷新と AI 活用ページ追加（2026-05-29〜31）

サイト（e2life.dev）のデザイン刷新から、AI 活用ページの追加までを一気通貫で進めたセッションの経緯。

## デザイン方向の選定と転換

- デザイン刷新の依頼を受け、**Claude Design**（claude.ai/design）で 4 方向（洗練ミニマル / 開発者ターミナル風 / ダーク AI モダン / エディトリアル）を生成・比較
- 最初に**開発者ターミナル風**を選定し、基盤 → トップまで実装してプレビュー提示
- ユーザー評価が「微妙」だったため**ダーク AI モダンへ方向転換**（ターミナル風の PR はクローズ）
- ダーク AI モダンで再度トップを実装。さらにバリエーションを生成し、最終的に「**D2 グリッド構造**」（罫線グリッド背景・番号付きセクション・ライト主役）を採用
- 配色は赤 → オレンジのグラデーション/グロー。架空の数値メトリクスは載せない方針

## 全ページ展開とマージ

- 基盤 → トップ → skills → career → contact を**機能単位の複数 PR（スタック）**で実装しマージ
- スタック PR のマージで `--delete-branch` を使い、base ブランチ削除により依存 PR が自動クローズされる事故が発生（リカバリ済み。詳細は learnings）

## AI 活用ページ

- `/ai/` ハブ + 詳細 6 ページを動的ルート + データ駆動で追加
- 詳細ページの内容は、別リポの実際の取り組みから**公開可能な技術的枠組みのみ**を抽出。workflow で並列抽出し、機密スクリーニング + 手動精査 + ユーザー確認の多層でガード

## 付随対応

- 冗長なセクションラベル（`// 強み` 等）を除去
- データ自動同期（外部 API 連携）を方針変更で廃止し、関連 docs を手動運用に統一

## 記録の置き場をめぐる議論

- セッション中、振り返りを誤って Claude Code 標準メモリ（`~/.claude/.../memory/`）に記録していた
- ユーザー指摘により、記録は **repo 内の record-layers**（conversations / decisions / learnings / claude-behavior）に取るのが正しいと確認（旧メモリ方式は廃止）
- 記録層を配備し、本セッションの振り返りを各層へ記録（本ファイルもその一環）
