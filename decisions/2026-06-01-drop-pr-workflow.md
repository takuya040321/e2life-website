# PR 必須運用を廃止し main 直 push 運用に

- 日付: 2026-06-01
- 対象: 開発フロー（git 運用 / CI トリガー前提 / CLAUDE.md の git・コード品質基準・開発フロー）

## 採用した方針

- feature ブランチ + PR 必須運用を廃止し、**main 直 push を許可** する
- PR description / closed PR 一覧での開発ストーリー記録は行わない
- 開発履歴は **commit log で辿る**。commit message は Conventional Commits 準拠を維持
- claude-assets の `rules/git.md` デフォルト（main 直 push 可）に揃え、本 repo の事業固有上書きを撤回

## 理由

- 個人 site で 1 人作業。レビュワーが存在せず、PR を作っても自分でマージするだけになる
- PR を介すコストが運用実効と見合わない（ブランチ切替・PR 作成・マージの 3 操作が毎回発生）
- Vercel preview URL の自動生成は機能としては有用だが、本 site の更新頻度・規模では必須要件ではない
- closed PR 一覧を「開発ストーリー」として読む運用を実際にはしていない。commit log で十分

## 却下した代替案と理由

- **PR 全面維持**: 1 人作業ではレビュー機能が働かず、形式コストのみ残る
- **一部例外運用**（共通インフラ変更だけ main 直 push 等）: 線引きが曖昧で運用が割れる。例外を判断する手間自体がコスト

## 影響範囲

- `CLAUDE.md` の git セクション（L48-52）/ コード品質基準（L62, L64）/ 開発フロー（L66-72）を main 直 push 運用に書き換え
- `.github/workflows/ci.yml` は当初 `pull_request` トリガーだったが、ユーザー裁定で本決定に含めることに変更（2026-06-01）。`on: push: branches: [main]` に書き換える

## 過去判断との関係

- CLAUDE.md が PR 必須の唯一の原本だった。本決定で CLAUDE.md を上書きすることで方針転換が確定する
- 既存 decisions ファイルでの言及はないため、過去 decisions の改変は行わない（追記型で本決定を新規記録）

## 見直しトリガー

- 共同編集者が増えた場合（レビュー機能が必要になる）
- Vercel preview URL を運用上必須にしたくなった場合（公開前確認フローが要件化した場合）
- レビューが必要な変更（破壊的変更・公開コンテンツの大幅改訂）が増えた場合
